# Dotfiles SWOT Analysis and Overhaul Plan

Date: 2026-07-06
Scope: `~/.local/share/chezmoi` (repo `imkulwant/dotfiles`, branch `master`)
Reviewer: Claude (Opus 4.7)

This document reviews the current dotfiles repo, does a SWOT analysis, calls out concrete bugs, and lists what is missing versus modern industry practice.
It is intended as the working brief for the planned overhaul.

## 1. What the repo currently is

A chezmoi-managed macOS dotfiles repo with:

- Shell: zsh + Oh My Zsh + Powerlevel10k, plus `zsh-autosuggestions` and `zsh-syntax-highlighting`.
- Package management: single `Brewfile` (formulae, casks, `vscode` extensions).
- Language toolchains: `openjdk@21`, `maven`, `jenv`, `python@3.13`, `pyenv`, `pipx`.
- Editors: VS Code (settings tracked at `private_Library/.../Code/User/settings.json`), IntelliJ IDEA.
- Bootstrap: `install.sh` (one-shot from a fresh Mac) + `.chezmoiscripts/run_once_before_*` scripts.
- External assets: `.chezmoiexternal.toml` pulls oh-my-zsh, powerlevel10k, and the two zsh plugins as tarballs.
- Aliases: `dot_aliases.zsh.tmpl` (mostly git shortcuts + a few ls/nav aliases).
- No CI, no tests, no secrets management, no `~/.gitconfig`, no `~/.ssh/config`, no tmux/nvim config.

## 2. SWOT

### 2.1 Strengths

- **Right tool for the job.** chezmoi is the correct choice: templates, run-once scripts, externals, encryption support, cross-machine.
- **Reproducible package baseline.** A single `Brewfile` captures formulae, casks, and VS Code extensions in one place.
- **Bootstrap exists.** A one-shot `install.sh` gets a new Mac from zero to usable.
- **Idempotent primitives.** `run_once_before_*` scripts use `command -v` / directory checks before installing.
- **Editor settings versioned.** VS Code `settings.json` is under source control, not floating in `~/Library`.
- **Alias hygiene.** Aliases are grouped and commented, which is rare.
- **README-as-cheatsheet.** The README doubles as a chezmoi quick-reference, which is genuinely useful.

### 2.2 Weaknesses (bugs and design gaps found in this pass)

Bugs that will bite on a fresh install:

1. **`BrewFile` vs `Brewfile` casing.** `run_onchange_install-brewfile.sh.tmpl:3` references `.../BrewFile`; the actual file is `Brewfile`. On any case-sensitive filesystem (APFS case-sensitive, Linux, CI) this fails silently or errors.
2. **`install.sh` runs `brew bundle install --file ~/Brewfile`**, but the Brewfile lives in `~/.local/share/chezmoi/Brewfile`, not `~/Brewfile`. The command will fail unless the user manually copied it.
3. **Duplicated bootstrap paths.** `install.sh` installs Homebrew, chezmoi, oh-my-zsh AND runs brew bundle; the `.chezmoiscripts/run_once_before_*` scripts do the same. Two sources of truth, ordering not obvious.
4. **Duplicated oh-my-zsh install.** `.chezmoiexternal.toml` downloads oh-my-zsh as an archive to `~/.oh-my-zsh`, but `run_once_before_30_install_ohmyzsh.sh.tmpl` also curls the installer. They will race and overwrite each other.
5. **Duplicated powerlevel10k / plugins install.** Same as (4): `.chezmoiexternal.toml` downloads them, `run_once_before_40_...` and `run_once_before_ohmyzsh-plugins.sh.tmpl` also git clone them.
6. **`.chezmoiignore` ignores `.oh-my-zsh/custom/*`** but `.chezmoiexternal.toml` writes plugins **into** `.oh-my-zsh/custom/plugins/*`. The two configs contradict each other.
7. **Java version drift.** `Brewfile` has `openjdk@21`; VS Code `settings.json` sets `JavaSE-24` (path `/opt/homebrew/opt/openjdk@24/...`) as default. `openjdk@24` isn't installed by this repo, so VS Code will fail to find it on a fresh machine.
8. **Python version drift.** `Brewfile` now has `python@3.13` (per commit `153e8d5`), but VS Code `settings.json` pins `python.defaultInterpreterPath` and `code-runner.executorMap` to `/opt/homebrew/bin/python3.12`. Broken after apply.
9. **`chsh -s /bin/zsh` uses system zsh** (Apple's, currently 5.9), not Homebrew's newer zsh, defeating the purpose of `brew "zsh"` in the Brewfile.
10. **`run_onchange_install-brewfile.sh.tmpl` is unguarded by `set -euo pipefail`.** So is every other script under `.chezmoiscripts/`. Any silent failure (network, brew not on PATH yet) leaves the machine half-configured with no signal.
11. **Aliases file contains stray citation markers** (`[1][2][3]`, `[5][3]`, etc.), left over from an LLM paste. Cosmetic, but embarrassing.
12. **`.chezmoidata.yaml` is a no-op.** It only sets `user: "{{ .chezmoi.user }}"` which is already available via `.chezmoi.username`. Nothing consumes it.
13. **`~/.gitconfig` not managed.** No git identity, no `pull.rebase`, no `init.defaultBranch`, no delta pager, no aliases at the git level (only shell aliases). Fresh machine has no name/email until user manually sets it.
14. **`~/.ssh/config` not managed.** Common host aliases and identity separation are re-created by hand on every new machine.
15. **Redundant Python managers.** `pyenv` + `pipx` + `python@3.13` from brew all coexist. Global user CLAUDE.md standardises on `uv`; the repo does not reflect that.
16. **`v='nvim'` alias but nvim is not in the Brewfile.** Alias is a landmine on a fresh machine.
17. **`~/.p10k.zsh` is 90KB and tracked verbatim.** It is regenerated by `p10k configure`; storing the full file is fine, but a comment explaining that + a link to the generator would save future confusion.
18. **Branch is `master`.** GitHub's default is `main` for over four years now. Cosmetic but consistently flagged by tooling and by linters.

Design gaps (not bugs, but weaknesses):

19. **No CI.** Nothing runs `shellcheck`, `shfmt`, `chezmoi verify`, or `brew bundle check` on push. Bugs 1-10 above would have been caught by any of these.
20. **No `pre-commit`.** Same story locally.
21. **No secrets story.** Any credential (github token, npm token, cloud creds) currently has to live outside the repo unmanaged. chezmoi supports age/1Password/gpg and none is wired up.
22. **No macOS defaults automation.** Dock, Finder-show-hidden, screenshot location, key-repeat, natural-scroll, etc. are all set by hand.
23. **No font management.** Powerlevel10k needs a Nerd Font (MesloLGS NF); nothing installs it. First run always looks broken until user opens p10k configure and follows the manual font instructions.
24. **No Mac App Store integration.** `mas` is not used, so App Store apps aren't captured.
25. **VS Code settings drift.** Managing raw `settings.json` conflicts with VS Code's own Settings Sync; either use one or the other, not both.
26. **No changelog / no versioning.** For a personal repo that's fine, but there is also no `LICENSE`, which `.chezmoiignore` already accounts for as if one existed.
27. **Documentation is a command cheatsheet, not a design doc.** A new machine (or future-you) has no explanation of what is intended to be here or why oh-my-zsh + p10k rather than starship + zinit.

### 2.3 Opportunities

- **Consolidate bootstrap into one path.** Delete `install.sh` OR delete the duplicated `.chezmoiscripts/`; make `chezmoi init --apply` the single entrypoint. Recommended: keep chezmoi scripts, replace `install.sh` with a 5-line curl-to-chezmoi bootstrap.
- **Adopt `mise` (or `asdf`) as the single version manager.** Replaces `jenv` + `pyenv` + any future `nvm`/`rbenv` with one tool and one config (`.tool-versions` or `mise.toml`). Cuts three brew formulae and two shell hooks.
- **Switch shell framework.** Options: `starship` prompt + plain zsh + `zinit` (fast, industry-common), or keep oh-my-zsh but only via the archive external (drop the shell script). Recommended: starship + zinit.
- **Modern CLI replacements.** Add `bat`, `eza`, `ripgrep`, `fd`, `zoxide`, `git-delta`, `dust`, `btop`, `tldr`, `atuin`. All well-maintained, all in brew, all give visible daily wins.
- **`gh` CLI.** Given the user already scripts against GitHub, `gh` should be in the Brewfile.
- **Tmux or Zellij config.** Even a minimal `~/.tmux.conf` with sensible defaults + `tpm` avoids the "first thing I do on a new machine" trap.
- **Neovim config.** Either a kickstart.nvim fork or LazyVim, tracked here. The `v='nvim'` alias currently promises something the repo doesn't deliver.
- **`~/.gitconfig` template.** Global identity, `init.defaultBranch = main`, `pull.rebase = true`, `push.autoSetupRemote = true`, `rerere.enabled = true`, delta as pager. This is one of the highest-ROI additions.
- **Secrets via chezmoi + 1Password (or `age`).** `chezmoi.io/user-guide/password-managers/1password/` — cleanest path for a solo user. Age is fine if there is no 1Password.
- **CI on push.** GitHub Actions matrix (macOS + linux-container) that runs `shellcheck`, `shfmt -d`, `chezmoi execute-template`, and `chezmoi apply --dry-run` against a scratch home. Catches every drift bug found in section 2.2.
- **`pre-commit` hooks.** Same checks locally, plus `end-of-file-fixer`, `trailing-whitespace`, `check-added-large-files`.
- **macOS `defaults` script.** A `.chezmoiscripts/run_onchange_macos_defaults.sh.tmpl` that applies opinionated defaults (Dock autohide, show hidden files, faster key-repeat, screenshot location, disable "correct spelling automatically", etc.).
- **Nerd Font install.** Add `homebrew/cask-fonts` tap + `cask "font-meslo-lg-nerd-font"` so p10k renders correctly on first run.
- **Split configs by machine.** chezmoi templates on `.chezmoi.hostname` / a `data.work` boolean, so a work Mac and a personal Mac diverge cleanly without a fork.
- **Auto-update.** launchd agent (or a `chezmoi update` in `zshrc` behind a "once per day" guard) so the machine stays in sync.
- **Match global CLAUDE.md standards.** The user's global `~/.claude/CLAUDE.md` mandates `uv` for Python. This repo still installs `pyenv` + `pipx`. Replace with `uv` (single brew formula) and delete the other two.
- **Rename `master` → `main`.**
- **Add a `LICENSE`.** MIT or similar. `.chezmoiignore` already assumes it.
- **`justfile` or `Makefile`.** A single `just apply`, `just verify`, `just doctor`, `just update` shortens the mental model.

### 2.4 Threats

- **Silent partial installs.** Because no script uses `set -euo pipefail`, a fresh install can succeed exit-code-wise while leaving the machine broken. Discovered only later, in isolation, without a repro.
- **Upstream tarballs are unpinned.** `.chezmoiexternal.toml` uses `master.tar.gz` for oh-my-zsh and the plugins. A breaking upstream change breaks every fresh install with no rollback.
- **Version drift between Brewfile and settings.json.** Already happening today with python@3.12→3.13 and openjdk@21 vs @24. Will keep happening without CI to check.
- **Powerlevel10k is in maintenance mode.** The maintainer paused active work. Still stable, but future macOS/zsh releases might break it and no one is patching quickly. Starship is the low-risk migration.
- **Oh My Zsh startup cost.** With three plugins + p10k it is measurably slow. Growth (more plugins, more work) will make it worse. Zinit / plain zsh solves this.
- **Secrets leak risk.** As soon as the user is tempted to check in a "just one" token, there is no encryption boundary set up. Better to install the boundary before the temptation.
- **`.DS_Store` files present in the source tree.** `.gitignore` catches new ones, but `private_Library/.DS_Store` and `.DS_Store` at repo root exist on disk (though not tracked). Any `chezmoi add` of a directory could accidentally pull them in.
- **iTerm2 telemetry / maintenance concerns.** iTerm2 is fine today, but Ghostty, WezTerm, and Kitty are all more actively developed. Worth a look before the next machine.

## 3. Missing pieces vs. common industry practice

Grouped by "how commonly you'd see this in a well-run 2026 dotfiles repo."

### 3.1 Table stakes (basically expected)

- `~/.gitconfig` template with identity, sane defaults, delta pager.
- `~/.ssh/config` template (with `IdentitiesOnly yes`, `UseKeychain yes` on macOS).
- CI running shellcheck + `chezmoi apply --dry-run`.
- `pre-commit` config.
- `set -euo pipefail` in every shell script.
- One version manager (`mise` or `asdf`), not three.
- A Nerd Font installed automatically if the prompt requires one.
- macOS `defaults write` script.
- `gh` CLI in the Brewfile.
- Modern coreutils replacements at least partially adopted (`bat`, `eza`, `rg`, `fd`).

### 3.2 Common (you'd see it on many staff-engineer dotfiles)

- `starship` prompt in place of p10k.
- `zoxide` + `atuin`.
- `git-delta`.
- `direnv` (or mise's built-in equivalent).
- `tmux` (+ tpm) or `zellij`.
- Neovim config (LazyVim / kickstart / personal).
- `just` as a task runner for the dotfiles themselves.
- Secrets via 1Password CLI or `age`.
- Per-host / per-OS templating.
- Some form of "run `chezmoi update` on login" auto-sync.
- LICENSE + a real README explaining the layout.

### 3.3 Nice-to-have (a smaller share of repos have these)

- Ghostty / WezTerm config in place of iTerm2 profile.
- `yazi` file manager.
- `k9s` + `kubectx` + `kubens` + `stern` (only k9s currently).
- `mas` for Mac App Store apps.
- Auto-update via launchd.
- Automated screenshots / dotbot-style demo GIF in README.
- Backup script (restic to B2 / iCloud, whatever).
- Per-language language servers pinned (already partially in VS Code).
- MCP / AI infra crosslinked into the shell env (given `~/.config/ai` exists).

## 4. Recommended overhaul plan (phased)

Order chosen so that each phase is independently useful and reversible.

### Phase 0 — Fixes (no design changes, half a day)

- Fix `BrewFile` casing bug.
- Fix `install.sh` Brewfile path.
- Delete one of the two oh-my-zsh install paths (keep the chezmoi script, drop from `.chezmoiexternal.toml`, or vice versa).
- Update VS Code `settings.json` to `python@3.13` and `openjdk@21` so it matches the Brewfile.
- Add `set -euo pipefail` to every script under `.chezmoiscripts/`.
- Strip citation markers from `dot_aliases.zsh.tmpl`.
- Add `LICENSE` (MIT).
- Rename branch `master` → `main`.

### Phase 1 — CI and hygiene (half a day)

- Add `pre-commit` config (shellcheck, shfmt, trailing-whitespace, end-of-file-fixer).
- Add GitHub Actions workflow: shellcheck + `chezmoi execute-template` on templated files + `chezmoi apply --dry-run` in a clean container.
- Move `install.sh` to a documented one-liner in the README (curl-to-chezmoi); delete `install.sh`.

### Phase 2 — Managed configs added (one day)

- `~/.gitconfig` (with delta pager).
- `~/.ssh/config` (macOS keychain + IdentitiesOnly).
- `~/.config/starship.toml` (replaces p10k).
- Nerd Font cask in Brewfile.
- `gh` in Brewfile.
- Modern CLI tools in Brewfile: `bat`, `eza`, `ripgrep`, `fd`, `zoxide`, `git-delta`, `tldr`, `atuin`.

### Phase 3 — Consolidation (one day)

- Replace `jenv` + `pyenv` + `pipx` with `mise`.
- Update aliases and shell init accordingly.
- Delete now-unused Brewfile lines.
- Update VS Code Java/Python runtime paths to `mise` shims.

### Phase 4 — Nice things (as time permits)

- Neovim config (LazyVim fork or kickstart.nvim).
- Tmux / Zellij config.
- macOS `defaults` script.
- Secrets via 1Password CLI (or `age`).
- Per-host template split (work vs. personal).
- `justfile` for `just apply / verify / doctor / update`.
- launchd agent for daily `chezmoi update`.

## 5. Success criteria for the overhaul

Before declaring the overhaul done, verify against this checklist (matches R4 in global CLAUDE.md):

- `chezmoi apply` on a scratch macOS VM succeeds and leaves an empty `chezmoi verify` and `chezmoi status`.
- Fresh shell starts in under 300ms.
- `brew bundle check --file $(chezmoi source-path)/Brewfile` is clean.
- `shellcheck` and `shfmt -d` pass on every `.sh` and `.tmpl` in CI.
- No file managed by chezmoi references a Homebrew formula that is not in the Brewfile (grep check in CI).
- VS Code opens a Java and a Python file with no "interpreter not found" prompts.
- `git config --get user.email` and `user.name` return values without manual setup.
- Prompt renders correctly (Nerd Font glyphs visible) on first launch.

## 6. Non-goals

To keep scope honest:

- Not migrating away from macOS or supporting Linux from this repo (though CI will use a Linux container for lint/dry-run).
- Not adopting NixOS / nix-darwin (bigger swing than requested).
- Not managing application data (only configs).
- Not building a public dotfiles project; this is a personal repo, docs written for future-me not for strangers.
